Informações gerais

Objetivo da oficina Esta oficina tem o objetivo de introduzir algumas ferramentas para pesquisadores da área de exatas que podem ajudar no desenvolvimento de sua pesquisa de um modo geral. Esta oficina segue o modelo das oficinas propostas pelas organizações internacionais Software Carpentry e Data Carpentry, responsáveis pela realização de dezenas de oficinas deste tipo em diversas cidades ao redor do mundo. O ganho para o participante é imediato, principalmente se sua pesquisa envolve o uso de computador para realização de tarefas repetitivas ou análise de dados.

Para mais informações sobre as motivações do Software Carpentry, veja o artigo "Best Practices for Scientific Computing".

Quem: O curso é direcionado a professores, pesquisadores e alunos de pós-graduação, ou de graduação avançados. Não é necessário nenhum conhecimento prévio sobre os softwares para participar.

Instalações

Em construção

Para participar de um workshop Software Carpentry, você precisa ter acesso aos softwares descritos abaixo. Além disso, você precisará de um browser atualizado.

Nós mantemos uma lista de problemas comuns que podem ocorrer durante a instalação, como uma referência para instrutores, e que pode ser encontrada na Página de problemas de configuração e soluções (em inglês).

O Shell Bash

O Bash é um shell muito utilizado, e que oferece o poder de realizar tarefas simples rapidamente.

Windows

Video Tutorial
  1. Download do instalador do Git para Windows.
  2. Rode o instalador e siga os passos abaixo:
    1. Clique "Next".
    2. Clique "Next".
    3. Clique "Next".
    4. Clique "Next".
    5. Clique "Next".
    6. Selecione "Use Git from the Windows Command Prompt" e clique "Next". Se você esquecer dessa etapa, os programas necessários para o workshop não irão funcionar. Se isso acontecer, rode novamente o instalador e selecione a opção apropriada.
    7. Clique "Next". Mantenha "Checkout Windows-style, commit Unix-style line endings" selecionado.
    8. Selecione "Use Windows' default console window" e clique "Next".
    9. Clique "Next".
    10. Clique em "Finish".

Estes passos irão fornecer tanto o Git quanto o Bash no programa Git Bash.

Mac OS X

O shell padrão em todas as versões do Mac OS X é o Bash, então não é necessário fazer a instalação. Você acessa o Bash a partir do Terminal (encontrado em /Applications/Utilities). Veja o video tutorial da instalação do Git para um exemplo de como abrir o Terminal. Você pode manter o Terminal no seu dock para este workshop.

Linux

O shell padrão geralmente é o Bash, mas se sua máquina possuir uma configuração diferente, você pode rodá-lo abrindo um terminal e digitando bash. Não há necessidade de instalação.

Git

Git é um sistema de controle de versão que permite rastrear quem fez modificações, aonde elas foram feitas e quando. Também possui opções para facilmente compartilhar uma versão pública do seu código no github.com. Você precisará de um browser suportado (atualmente Chrome, Firefox ou Safari, ou Internet Explorer 9 ou acima).

Windows

Git deve estar instalado no seu computador como parte da instalação do Bash (descrito acima).

Mac OS X

Video Tutorial

Para OS X 10.9 e superior, instale Git para Mac baixando e rodando a versão mais recente do instalador "mavericks" a partir desta lista. Após instalar o Git, não haverá nada no seu folder /Applications, pois o Git é um programa de linha de comando. Para versões mais antigas do OS X (10.5-10.8) use o instalador mais recente denominado "snow-leopard" disponível aqui.

Linux

Se o Git ainda não estiver disponível em sua máquina, você pode tentar instalar através do gerenciado de pacotes da sua distribuição. Por exemplo, para Debian/Ubuntu rode sudo apt-get install git e para Fedora rode sudo yum install git.

Editor de Texto

Quando você está escrevendo código, é importante usar um editor de texto que seja otimizado para escrever código, com características como coloração automática de palavras-chave. O editor de texto padrão no Mac OS X e Linux geralmente é o Vim, que não é famoso por ser intuitivo. Se você acidentalmente se encontrar preso nele, tente digitar a tecla Esc, seguido de :q! (dois pontos, 'q' minusculo, ponto de exclamação), depois aperte Enter para retornar ao Shell.

R
